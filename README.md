# Book Reader

A Flutter Android book reader supporting **EPUB, PDF, TXT, AZW, AZW3,
and MOBI**, with paginated reading, custom annotations (citations,
dictionaries, characters), and a series-organized library.

Kindle formats (AZW / AZW3 / MOBI) are converted on the fly via the
sister project [`kindle_unpack`](https://github.com/emaurel/kindle_unpack)
— a pure-Dart port of KindleUnpack — and then read like any EPUB.

## Prerequisites

- Flutter 3.22+ (`flutter --version`)
- Android SDK with API 26+
- A device or emulator running Android 8.0 or higher

## First-time setup

This repo contains the `lib/`, `android/app/src/main/AndroidManifest.xml`,
`pubspec.yaml`, and config files. Platform scaffolding is generated by
Flutter itself. From an empty folder:

```bash
# 1. Initialize the Flutter project (only needed once per clone if
#    android/ios folders are missing)
flutter create --org com.yourname --project-name book_reader --platforms=android .

# 2. Pull dependencies
flutter pub get

# 3. Replace the auto-generated AndroidManifest.xml with the one in
#    this repo (it has the storage permissions we need)

# 4. Run on a connected device
flutter run
```

If the `android/` folder is already complete after cloning, just:

```bash
flutter pub get
flutter run
```

## Features

### Library
- **Series-grouped grid** — books are arranged under their series in
  collapsible groups, sorted by `seriesNumber` ascending. Books without
  a series fall under "Other" at the bottom.
- **Continue reading** — a drawer entry above Library showing only
  books with progress > 0 and < 1, ordered by last-opened.
- **Metadata editor** — long-press a book → "Edit info" lets you
  change title, author, series, series number, and description.
  Empty fields persist as `NULL` (so you can actually clear a series).
- **Book info sheet** — long-press → "Book info" shows the cover,
  metadata, file size, dates, progress, and description.
- **Device scan** — walk `/sdcard/Download`, `Documents`, `Books`,
  `eBooks` for `.epub` / `.pdf` / `.txt` / `.azw[3]` / `.mobi` files
  using `MANAGE_EXTERNAL_STORAGE` access. Dedupes by file path AND
  pre-conversion source path so re-scans don't re-convert the same
  Kindle file.
- **Show documents toggle** (Settings → Library) — when off, hides PDF
  and TXT files in both the library view and the device scanner.
- **Refresh covers & metadata** — a one-tap action that re-extracts
  cover + author/series/description for any book that's missing them.
- **File picker import** with copy-to-app-private-storage for stable
  file paths.
- **Cover extraction**:
  - EPUB — manifest walk: `properties="cover-image"` → `<meta
    name="cover">` → filename heuristic → largest-image fallback.
  - PDF — page 1 rendered as JPEG via `pdfx`.
  - AZW / MOBI — extracted from the EPUB produced by `kindle_unpack`.

### Formats
- **EPUB** — paginated WebView with CSS multi-column layout. One
  screen = one page, no scrolling.
- **PDF** — paged rendering via `pdfx`, horizontal scroll.
- **TXT** — custom paginator using `TextPainter` measurement, paged.
- **AZW / AZW3 / MOBI** — converted to EPUB at import/scan time via
  `kindle_unpack` and treated as EPUB thereafter. Failed conversions
  show a "convert via Calibre" placeholder.

### Reader
- **Page-mode** — left-third tap or right-swipe → previous page,
  right-third tap or left-swipe → next page, middle tap → toggle chrome.
- **Page-turn animation** — body-translate slide; forward goes
  leftwards, backward goes rightwards (clear directional cue).
- **Chapter picker** — list icon in the AppBar opens a bottom sheet
  with every chapter (sub-chapters flattened, current one highlighted,
  list auto-scrolls to land near it). Falls back to the first
  `<h1>`/`<h2>`/`<h3>` heading when the NCX label is generic
  (e.g. "Part 5"), common with `kindle_unpack`-converted files.
- **Reader settings** (slider sheet) — font family (Inter / Lora /
  Merriweather / Source Serif 4), font size, line height, horizontal
  margin, theme (light / sepia / dark / black), keep-screen-on.
- **Bottom progress bar** — shows fraction-done, current page within
  chapter, and total chapters. Themed to the reader palette.
- **Immersive mode** — when chrome is hidden, the Android bottom nav
  bar hides too. Restored on reader exit.
- **Citation preview** — tapping a citation in the Citations screen
  opens the reader at the citation's chapter+page **without** updating
  the saved reading position.

### Annotations (selection menu)
Long-press a word, then drag the orange handles to extend the
selection across phrases or paragraphs. The selection menu appears
above the highlight with these actions (modular registry — adding a new
one is one line in `lib/features/reader/selection/selection_actions_provider.dart`):

- **Copy** — clipboard.
- **Citation** — saves the passage with chapter + character offsets so
  the highlight can be re-painted on later opens. Listed in the
  Citations drawer screen; tap any entry to preview-jump to the
  reader at its location. Tap an in-reader citation to remove it.
- **Dictionary** — pick (or create inline) a user dictionary,
  optionally scoped to the current book's series, then attach a
  definition. Every standalone occurrence of the word in any
  matching book gets a blue dotted underline. Tap to view / edit /
  delete the entry.
- **Character** — pick (or create) a character, optionally
  series-scoped. Save the selected text as a description. Every
  occurrence of the character's name (or any of its aliases) gets a
  green dashed underline. Tap to view all saved descriptions; manage
  aliases inline.

### Drawer
- **Continue reading** — books in progress (most recently opened first).
- **Library** — series-grouped grid.
- **Citations** — every saved citation, source book + date.
- **Dictionaries** — per-dictionary entries, with optional series scope.
- **Characters** — grouped by series; expand a character for aliases +
  saved descriptions.
- **Settings** — theme, library toggles.

### Storage
- SQLite via `sqflite`.
- Schema migrations through **v9**, idempotent (recoverable from
  partial migrations using `IF NOT EXISTS` and `try/catch ALTER`).
- `SharedPreferences` for theme + reader settings.

### State
- [Riverpod 2.x](https://riverpod.dev) — providers live next to the
  feature they belong to. A handful of `revisionProvider`s
  (`dictionaryEntriesRevisionProvider`, `characterRevisionProvider`)
  drive automatic re-paint of in-reader highlights when the
  underlying data changes anywhere in the app.

## Project structure

```
lib/
├── main.dart                      # Entry point
├── app.dart                       # Root widget, theme, router
├── core/
│   ├── database/                  # SQLite + migrations (v9)
│   ├── router/                    # go_router config
│   └── theme/                     # App + reader themes
├── features/
│   ├── library/                   # Library, Continue reading, scan,
│   │                              #   import, metadata editor
│   ├── reader/                    # EPUB / PDF / TXT / AZW viewers
│   │   ├── presentation/          #   reader screen, popups, picker
│   │   ├── providers/             #   reader controls, progress
│   │   └── selection/             #   modular SelectionAction registry
│   ├── citations/                 # Saved passages, persistent highlight
│   ├── dictionary/                # User-defined dictionaries + entries
│   ├── characters/                # Characters + aliases + descriptions
│   └── settings/                  # User preferences
└── shared/
    └── navigation/                # Drawer entries (single source of truth)
```

## Notable dependencies

- [`kindle_unpack`](https://pub.dev/packages/kindle_unpack) — sister
  project, pure-Dart MOBI / AZW3 / KF8 reader.
- [`epubx`](https://pub.dev/packages/epubx) — EPUB metadata + chapter
  parsing.
- [`pdfx`](https://pub.dev/packages/pdfx) — PDF rendering + page-1
  cover thumbnails.
- [`webview_flutter`](https://pub.dev/packages/webview_flutter) — used
  for the EPUB reader's CSS-column page layout.
- [`go_router`](https://pub.dev/packages/go_router) — routing.
- [`flutter_riverpod`](https://pub.dev/packages/flutter_riverpod) —
  state management.
- [`sqflite`](https://pub.dev/packages/sqflite) — SQLite.

## Roadmap (v2 ideas)

- Pre-paginate every chapter at startup for exact "page X / Y" totals
  across the book (current display is per-chapter only).
- Embedded image rendering inside the EPUB WebView (currently shows
  broken icons; the bytes live inside the EPUB zip).
- Real AZW3 chapter titles in the chapter picker — currently relies
  on a heading-text heuristic when `kindle_unpack` emits "Part N".
- TXT and PDF text selection (PDF rendered as image, no text layer).
- Full-text search.
- OPDS catalog browsing.
- Text-to-speech.
